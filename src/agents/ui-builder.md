## Quality Standards

### Must Have
- TypeScript strict mode (no any)
- shadcn/ui components
- Tailwind utility classes only
- Thai mock data
- Mobile-first responsive
- Loading, empty, error states

### Must NOT Have
- Inline styles
- Hardcoded colors
- Lorem ipsum text
- Console.log statements
- Unused imports
- Any type assertions
